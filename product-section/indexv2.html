TagSafe · Tag validator

Compress third-party tag QA into one screen

Browser-based validator for third-party tags with live previews and automated checks.

Third-party tags used to be QA’d with a mix of dev tools, test pages, and manual notes, which made each creative slow to validate and easy to break when vendors sent updates.

  • Problem: Manual tag QA took 15–20 minutes per creative, spread across multiple tools. HTTPS, click tracking, and structure checks were easy to miss under time pressure.
  • Solution: Built a single-page validator that accepts pasted tags, runs core checks (structure, HTTPS, basic dimension sanity, click URL presence), and renders a live preview in the browser.
  • Impact: Collapsed QA into one screen and made it straightforward to re-run checks whenever a tag changes, cutting personal QA time substantially.

Outline · Image border tool

Make white-edge creatives compliant in seconds

Used by account managers, ad ops, and our trafficking vendor at the Post.

Many client assets arrive with white edges. Policy requires a visible border so ads aren’t confused with article imagery, but updating each size manually slows launches.

  • Problem: Adding borders meant re-requesting assets or opening each creative in Photoshop/Canva. For several concepts across 6+ sizes, that easily became a 10–20 minute chore per campaign.
  • Solution: Built a browser tool where anyone can upload standard display assets, apply a policy-compliant border in one click, and download renamed files that clearly indicate the border has been added.
  • Impact: Turned a license-dependent, ~1-minute-per-asset step into a ~15-second process for the full creative set. A typical once-a-week border job now takes under a minute instead of up to 20.

AdBuilder · Layout engine

Map one asset set across multiple display sizes

Internal tool to visualize headline/logo/CTA layouts across common formats.

Display campaigns often share the same core assets, but designers and ad ops see each size in isolation, which makes inconsistencies and cramped layouts harder to catch.

  • Problem: Reviewing each size separately slowed QA and made it easy to ship creatives where key elements didn’t translate well between formats.
  • Solution: Built a layout engine that takes a shared set of assets (logo, image, headline, body, CTA) and renders them into predefined sizes for side-by-side evaluation.
  • Impact: Makes it faster to spot layout issues across a campaign and reduces back-and-forth with designers before trafficking.

Vast2MP4 · Video utility

See what’s inside a VAST tag at a glance

Parses VAST XML to surface media URLs and trackers for faster debugging.

Video issues often trace back to specific files or trackers inside a VAST tag, but inspecting them by hand in raw XML is slow and brittle.

  • Problem: Finding the right MP4 URL or tracker in a VAST response meant digging through nested XML or relying on dev tools every time an issue came up.
  • Solution: Built a parser that accepts a VAST tag, then lists out media files, key metadata, and impression/click trackers in a human-readable way.
  • Impact: Speeds up troubleshooting and makes it easier to confirm file sizes, URLs, and tracker coverage without touching dev tools.

PageID Scraper · CMS helper

Bridge CMS URLs and ad server targeting

Turns WaPo article URLs into page IDs for faster targeting setup.

Page-level targeting often relies on internal IDs, not the public URLs that most stakeholders share when requesting campaigns.

  • Problem: Converting a list of article URLs into page IDs required repeated lookups in the CMS, which was slow and prone to copy/paste errors.
  • Solution: Built a small tool that accepts article URLs, queries for their page IDs, and returns a clean list ready for ad server targeting.
  • Impact: Shrinks a multi-step CMS task down to a single paste-and-download step, and standardizes how page IDs are pulled for campaigns.

BlockList · Brand safety

Turn messy URLs into usable blocklists

Helps translate flagged pages into exclusion keywords for campaigns.

When content is flagged as unsafe, ad ops often receives URLs, not the structured exclusion lists that platforms actually need.

  • Problem: Converting long lists of URLs into meaningful keywords was a manual, spreadsheet-heavy task that varied by who did it and took time away from higher-value work.
  • Solution: Built a helper that ingests URLs, extracts key terms, deduplicates them, and outputs a list ready for key-value or platform exclusions.
  • Impact: Makes blocklist creation faster and more consistent, especially when large spreadsheets of flagged URLs come in.

Collate · Spreadsheet helper

Convert a column into a clean list in one click

Tiny helper for turning spreadsheet exports into targeting strings.

A lot of key-value setup starts with a CSV export. Turning a column into a comma-separated list is simple, but repetitive and easy to mangle.

  • Problem: Manually joining long columns into a single string meant trimming spaces, hunting duplicates, and hoping no stray characters slipped in.
  • Solution: Collate cleans a column, removes duplicates, and joins it into one comma-separated value ready to paste into a UI or config file.
  • Impact: Turns a small but frequent spreadsheet chore into a one-click step and reduces the chance of subtle formatting errors.
